# Guia de Contribuição

## Escopo

Este é um repositório pessoal de estudos acadêmicos. Contribuições são bem-vindas, porém seguem diretrizes específicas.

## Como Contribuir

### 1. Issues

- Reporte problemas encontrados em projetos ou documentação
- - Sugira melhorias na organização do repositório
  - - Corrija erros em documentação ou código
   
    - ### 2. Pull Requests
   
    - Antes de submeter um PR:
    - - Faça um Fork do repositório
      - - Crie uma branch com nome descritivo: `feature/descricao` ou `fix/descricao`
        - - Mantenha o código limpo e bem documentado
          - - Adicione mensagens de commit claras
           
            - ### 3. Padrões de Código
           
            - - Use indentação consistente (2 ou 4 espaços)
              - - Escreva comentários em português
                - - Siga convenções de nomenclatura da linguagem utilizada
                 
                  - ### 4. Organização
                 
                  - Mantenha os arquivos organizados conforme a estrutura do repositório:
                  - - Projetos em `Projetos/` com subpastas por linguagem
                    - - Estudos em `Estudos/`
                      - - Documentação em `Recursos/`
                       
                        - ## Padrões de Commit
                       
                        - Use commits bem estruturados:
                       
                        - ```
                          <tipo>: <descrição breve>

                          <descrição detalhada se necessário>
                          ```

                          Tipos sugeridos:
                          - `feat`: Novo projeto ou funcionalidade
                          - - `fix`: Correção de bugs
                            - - `docs`: Alterações em documentação
                              - - `refactor`: Reorganização de código
                                - - `test`: Adição de testes
                                  - - `chore`: Tarefas de manutenção
                                   
                                    - ## Código de Conduta
                                   
                                    - - Respeite outros colaboradores
                                      - - Mantenha discussões profissionais e construtivas
                                        - - Não compartilhe informações sensíveis ou credenciais
                                         
                                          - ## Licença
                                         
                                          - Ao contribuir, você concorda que suas contribuições serão licenciadas sob a mesma licença MIT do projeto.
